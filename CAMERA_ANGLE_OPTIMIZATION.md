# 3D视角优化说明

## 🎯 优化目标

改善3D场景的初始展示角度，让节点分布更立体、更有纵深感，避免"扁平"视觉效果。

## 📷 视角改进

### 优化前（正面平视）
```tsx
position={[0, 0, 25]}  // 相机在Z轴正前方，完全正视
fov={60}               // 视角60度
```

**问题**：
- ❌ 完全正面视角，缺乏立体感
- ❌ 节点在Y轴和X轴上重叠，难以区分前后关系
- ❌ 看起来像"2D平面图"而不是3D空间

### 优化后（斜角俯视）
```tsx
position={[15, 12, 20]}  // 斜45度角俯视
fov={65}                 // 视角略微扩大
target={[0, 0, 0]}       // 看向场景中心
```

**改进**：
- ✅ 斜45度角视角，展现真实3D纵深
- ✅ 俯视角度让上下层次清晰可见
- ✅ X、Y、Z三个维度都有体现
- ✅ 类似游戏/3D软件的经典视角

## 🎨 视觉效果对比

### 相机位置分析

**优化前** `[0, 0, 25]`：
- X=0：正中间（无横向偏移）
- Y=0：水平视线（无上下偏移）
- Z=25：距离场景25单位

→ 结果：完全正面平视，像看"照片"

**优化后** `[15, 12, 20]`：
- X=15：右侧偏移（横向角度）
- Y=12：上方偏移（俯视角度）
- Z=20：距离场景20单位（略微拉近）

→ 结果：斜角俯视，像看"立体模型"

### 视野角度（FOV）

- **优化前**：60° - 标准视角
- **优化后**：65° - 略微扩大，能看到更多节点

FOV增加让场景"更开阔"，配合斜角度让立体感更强。

## 🔧 其他优化

### 1. 灯光增强

**优化前**（基础照明）：
```tsx
<ambientLight intensity={0.3} />
<pointLight position={[10, 10, 10]} intensity={1} />
<pointLight position={[-10, -10, -10]} intensity={0.5} color="#4A90E2" />
```

**优化后**（多角度立体照明）：
```tsx
<ambientLight intensity={0.4} />                              // 环境光提亮
<pointLight position={[15, 15, 15]} intensity={1.2} />        // 主光源（右上前）
<pointLight position={[-10, 5, -10]} intensity={0.6} color="#4A90E2" />  // 补光（左上后，蓝色）
<pointLight position={[0, -8, 10]} intensity={0.4} color="#9370DB" />    // 底部补光（紫色）
```

**改进**：
- 环境光从0.3提升到0.4，整体更亮
- 主光源强度1.2，位置配合相机角度
- 三点照明法：主光+左侧补光+底部补光
- 底部紫色光增加色彩层次

### 2. 相机控制优化

**新增限制**：
```tsx
target={[0, 0, 0]}                    // 始终看向场景中心
maxPolarAngle={Math.PI * 0.85}        // 限制向下旋转（不超过153度）
minPolarAngle={Math.PI * 0.15}        // 限制向上旋转（不低于27度）
minDistance={15}                       // 最近15单位（原来10）
maxDistance={50}                       // 最远50单位（原来80）
```

**好处**：
- ✅ 防止相机翻转到场景底部或顶部（避免"倒立"视角）
- ✅ 限制缩放范围，避免离得太近（看不清全局）或太远（看不清细节）
- ✅ `target={[0, 0, 0]}`确保旋转时始终围绕场景中心

### 3. 角度限制说明

**极坐标角（Polar Angle）**：
- 0° = 正上方（天顶）
- 90° = 水平线
- 180° = 正下方（地面）

**限制范围**：
- `minPolarAngle = 27°`（0.15π）：不能看得太高
- `maxPolarAngle = 153°`（0.85π）：不能看得太低

→ 保持在27°到153°之间，避免极端视角

## 📐 几何关系

### 相机到原点的距离

优化后相机位置`[15, 12, 20]`：
- 距离 = √(15² + 12² + 20²) = √(225 + 144 + 400) = √769 ≈ **27.7单位**

比优化前的25单位稍远，但因为斜角度，视觉上反而显得更"包围"。

### 俯视角度

垂直角度 = arctan(12 / √(15² + 20²)) = arctan(12 / 25) ≈ **25.6°**

这是一个舒适的俯视角度，类似：
- 游戏中的等距视角（Isometric）
- 建筑效果图的鸟瞰图
- 3D软件（Blender、Maya）的默认视角

### 水平角度

水平偏移角 = arctan(15 / 20) ≈ **36.9°**

右侧偏移约37度，让三维空间的X轴深度清晰可见。

## 🌌 场景布局优化建议

如果节点仍然看起来"扁"，可能是节点的Z轴坐标范围太小。检查`initial-graph.json`或节点生成逻辑，确保：

1. **Z轴分布范围**：节点的Z坐标应该在-10到+10之间分布
2. **球面布局**：使用斐波那契球面分布（已实现）
3. **核心节点**：Innovation Core在(0, 0, 0)
4. **子星系**：三个星系分散在球面上
