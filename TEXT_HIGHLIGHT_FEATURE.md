# 节点文字悬浮高亮功能（优化版）

## ✨ 功能说明

当鼠标悬浮在3D节点上时，节点的文字标签会放大并变为节点主题色，提供清晰的视觉反馈。

## 🎨 视觉效果

### 正常状态（未悬浮）
- **文字颜色**：白色
- **文字大小**：0.3
- **文字不透明度**：0.95（轻微透明）
- **描边宽度**：0.015（细描边）
- **描边颜色**：黑色（固定）

### 悬浮状态（hovered）
- **文字颜色**：节点主题色（动态，如蓝色、绿色、紫色等）✨
- **文字大小**：0.36（放大 20%）✨
- **文字不透明度**：1.0（完全不透明，更清晰）✨
- **描边宽度**：0.015（保持不变，避免模糊）
- **描边颜色**：黑色（保持不变，确保对比度）

## 🔧 设计优化说明

### v1（初版 - 已废弃）
❌ **问题**：悬浮时描边加粗+变色，导致文字模糊难以阅读
```tsx
outlineWidth={hovered ? 0.03 : 0.02}               // 描边加粗
outlineColor={hovered ? node.color : '#000000'}    // 描边变色
outlineOpacity={hovered ? 0.8 : 1}                 // 描边透明
```

### v2（优化版 - 当前）
✅ **改进**：简化效果，专注于清晰度和颜色变化
```tsx
fontSize={hovered ? 0.36 : 0.3}          // 放大20%
color={hovered ? node.color : 'white'}   // 颜色高亮
outlineWidth={0.015}                     // 固定细描边
outlineColor="#000000"                   // 固定黑色描边（对比度）
fillOpacity={hovered ? 1 : 0.95}         // 悬浮时完全不透明
```

## 🎯 设计原则

### 1. 清晰度优先
- **固定细描边**（0.015）：提供文字与背景的基础对比度
- **黑色描边**：无论文字什么颜色，黑色描边都能保证可读性
- **完全不透明**（fillOpacity: 1）：悬浮时文字更实在、更清晰

### 2. 简洁有效
- 只改变颜色和大小，避免过多视觉干扰
- 描边保持不变，减少视觉噪音
- 专注于核心反馈：颜色 = 识别，大小 = 强调

### 3. 性能考虑
- 减少动态属性变化（只有fontSize、color、fillOpacity）
- 固定的描边减少渲染开销

## 📝 实现代码

**文件**: `src/components/galaxy/Planet.tsx`

```tsx
<Text
  position={[0, node.size * 0.8, 0]}
  fontSize={hovered ? 0.36 : 0.3}          // 悬浮时放大20%
  color={hovered ? node.color : 'white'}   // 悬浮时节点色
  anchorX="center"
  anchorY="middle"
  outlineWidth={0.015}                     // 固定细描边
  outlineColor="#000000"                   // 固定黑色描边
  fillOpacity={hovered ? 1 : 0.95}         // 悬浮时更不透明
>
  {node.label}
</Text>
```

## 📝 技术细节

### fillOpacity vs opacity
- `fillOpacity`：控制文字填充的透明度（不影响描边）
- `opacity`：控制整个文字（包括描边）的透明度

使用`fillOpacity`可以让悬浮时文字更实在，但描边保持清晰。

### outlineWidth 选择
- `0.01`：太细，在远距离几乎看不见
- `0.015`：✅ 最佳平衡，提供足够对比度但不干扰
- `0.02`：稍粗，开始影响文字清晰度
- `0.03+`：过粗，文字变模糊

### fontSize 比例
- 节点放大：20%（1.0 → 1.2）
- 文字放大：20%（0.3 → 0.36）
- 保持一致的放大比例，视觉更协调
