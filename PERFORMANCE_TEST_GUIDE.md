## 测试1: LLM缓存功能

### 目标
验证相同参数的请求会命中缓存，响应时间从60秒降至<100ms

### 步骤
1. 双击3D节点添加关键词：`AI`, `医疗`, `智能诊断`
2. 点击"Surprise Me"按钮
3. **记录时间**: 首次响应时间约 _____ 秒
4. 查看Console，应该看到:
   ```
   💾 Cache miss for key: ...
   🚀 Calling LLM API: ...
   ✅ API Response Status: 200
   💾 Cached response for key: ... (1/100)
   ```
5. 等待3个idea显示完成
6. 点击"重新生成创意"按钮（会清空ideas）
7. 再次点击"Surprise Me"（相同关键词）
8. **记录时间**: 第二次响应时间约 _____ 毫秒
9. 查看Console，应该看到:
   ```
   ✅ Cache hit for key: ...
   ⚡ Cache hit, returning cached response
   ```

### 预期结果
- ✅ 首次调用：~8秒
- ✅ 缓存命中：<100ms
- ✅ Console显示缓存命中日志
- ✅ 生成的3个idea与首次相同

## 测试2: 并发请求控制

### 目标
验证同时最多只有3个请求在执行，超出的请求会排队

### 步骤
1. 准备5组不同的关键词组合:
   - 组1: `AI`, `医疗`, `诊断`
   - 组2: `区块链`, `金融`, `支付`
   - 组3: `物联网`, `智能家居`, `安全`
   - 组4: `AR`, `教育`, `互动`
   - 组5: `5G`, `车联网`, `自动驾驶`

2. 快速连续执行:
   - 添加组1关键词 → 点击"Surprise Me"
   - 立即删除关键词，添加组2 → 点击"Surprise Me"
   - 立即删除关键词，添加组3 → 点击"Surprise Me"
   - 立即删除关键词，添加组4 → 点击"Surprise Me"
   - 立即删除关键词，添加组5 → 点击"Surprise Me"

3. 观察Console输出，应该看到类似:
   ```
   🚦 ConcurrencyController initialized with limit: 3
   🏃 Task started (running: 1/3)
   🏃 Task started (running: 2/3)
   🏃 Task started (running: 3/3)
   ⏳ Task queued (running: 3/3, queue: 1)
   ⏳ Task queued (running: 3/3, queue: 2)
   ✅ Task completed (running: 2/3)
   📤 Processing queued task (queue remaining: 1)
   ```

### 预期结果
- ✅ 同时最多3个"running"
- ✅ 超出请求显示"queued"
- ✅ 完成后自动处理队列
- ✅ 所有请求最终都完成

## 测试3: 进度百分比显示

### 目标
验证Deep Analysis过程中显示实时进度和预估时间

### 步骤
1. 选择任意关键词生成3个idea
2. 选中1个idea
3. 点击"Deep Analysis"按钮
4. 观察进度界面:

   **初始状态** (0-5秒)
   - 进度条: 0% → ~12%
   - 倒计时: 40秒 → 35秒
   - Agent状态:
     - ⟲ 商业可行性评估 ~5s
     - ○ 技术架构设计 ~5s
     - ○ 生成报告摘要 ~30s

   **PM Agent完成** (5-10秒)
   - 进度条: ~12% → ~25%
   - 倒计时: 35秒 → 30秒
   - Agent状态:
     - ✓ 商业可行性评估
     - ⟲ 技术架构设计 ~5s
     - ○ 生成报告摘要 ~30s

   **Tech Agent完成** (10-15秒)
   - 进度条: ~25% → ~40%
   - 倒计时: 30秒 → 25秒
   - Agent状态:
     - ✓ 商业可行性评估
     - ✓ 技术架构设计
     - ⟲ 生成报告摘要 ~30s

   **Orchestrator进行中** (15-45秒)
   - 进度条: ~40% → ~95%
   - 倒计时: 25秒 → 0秒
   - 报告开始流式显示

### 预期结果
- ✅ 进度条从0%平滑增长到95%
- ✅ 倒计时从40秒递减到0秒
- ✅ Agent状态按顺序变化：○ → ⟲ → ✓
- ✅ 各Agent显示预估时间
- ✅ 进度条有渐变色动画
- ✅ 不会卡在100%

### 实际结果
- 进度条增长: □ 平滑 □ 跳跃
- 倒计时准确性: □ 准确 □ 偏差大
- Agent状态: □ 正确 □ 错误
- 预估时间显示: □ 显示 □ 未显示
